Note: The Database Link of Oracle is supported in PostgreSQL only if FDW (foreign data wrapper) is installed manually. Note: Oracle creates explicit indexes on primary key whereas PostgreSQL creates those indexes with a self-explanatory names such as Employee_Pkey, Department_Pkey. The name of the Oracle system created indexes will not match with PosgreSQL indexes. If there is some system defined indexes in Oracle, so after conversion the PostgreSQL will have it with different name. Note: For Synonyms, only storage objects are converted from source Oracle database schema to target PostgreSQL database schema and code objects are not converted. Note: Index partition count will differ, because if in Oracle any master table contains indexes and table has some partitions, in PostgreSQL by default index will be created on each partition (index partition) Note: PostgreSQL does not support Range with Interval partition of Oracle. As part of migration processes to handle existing Range with Interval partition, default partition table are created in PostgreSQL, so the Partition count of Source Oracle database and Target PostgreSQL will not match. Note: Oracle count of Package will not match with PostgreSQL because both package Procedure and Functions are converted into Functions in PostgreSQL. Note: Please manually convert Special Objects.
